## Introduction

Sans Paper Form is an advanced form builder that empowers users to create dynamic, professional forms in minutes. With its cutting-edge AI image tagging tools, it simplifies data organization and enhances productivity, making it the perfect solution for businesses, event managers, and creatives. # Features Overview
The app consists of three main screens: **Profile**, **Forms**, and **History**.

### Profile
This section displays your name, the current app version, and provides key functionalities such as:
- Sending invites
- Switch organization
- Switch role
- Switch team
- Accessing the privacy policy
- Logging out

### Forms
The **Forms** module in the Sans Paper Form system enables users to create, manage, submit, and export forms across various roles and platforms (web and mobile). This module is crucial for collecting structured data, ensuring compliance, and maintaining organization-level workflows.

#### 🔍 What Users Can Do

- **View Available Forms**  
  All forms available for the logged-in user should be displayed in the Form tab.

- **Submit Forms**  
  - User can fill and submit answers to forms.
  - A success popup is shown after submission.
  - Redirects back to the form overview screen.

- **Save Forms as Draft**  
  - User can save partially filled forms.
  - Drafts can be edited and re-saved or submitted.

- **Edit Draft Forms**  
  - Drafts are editable.
  - On submission, the draft version is removed.

#### 📜 How to Create a Subform

Follow these steps to set up a subform:

###### **Step 1: Setup on the Web**
1. Log in to your account at [SansPaper Forms](https://form.sanspaper.com/).
2. Navigate to the **Forms** tab in the left sidebar menu.
3. Click on **Form Templates**.
4. Select a form where you want to set up the subform.
5. Click the arrow next to the form and select **Edit**.
6. Choose the field where you want to set up the subform.
7. Click the **Properties** icon.
8. Go to the **Event** tab and click the **Add New** button.
9. Fill in the required details:
   - **Name**: Enter a name for the event.
   - **Effects**: Select **Open Subform** (currently the only working effect).
10. Complete all the fields in the **Settings** section.
11. Click **Save**.

---

###### **Step 2: Testing on Mobile**
1. Open the **SPForm** mobile app.
2. Select the form where you created the subform event.
3. Enter the value that you set in the designated field.
4. A pop-up should appear, informing you that a subform needs to be filled in at the end of the current form.
5. Submit the current form.
6. The app will navigate you to the subform.
7. Fill in the subform and submit it.
8. You can verify submission by checking the **History** tab, where two forms should now be recorded.

**Note:** If the subform is not submitted, the first form will not be submitted either.

### History
The **History** screen allows users to view, manage, and interact with forms they have either submitted or saved as drafts.

#### 🔍 What users can do

- Users can **view a list of forms** they have either:
  - **Submitted** (read-only)
  - **Saved as drafts** (editable)

- Forms are displayed in **descending order** (most recent first)

- If no forms are present (submitted or drafts), a **placeholder UI** is shown:
  - E.g., "You have no form submitted here yet."

- Users can **filter** between:
  - Submitted forms
  - Draft forms

- Users can **search forms** by:
  - Title
  - Date
  - Status (draft/submitted)

- Users can **edit draft forms**
  - Drafts can be resubmitted or updated

- Users can **view submitted forms**
  - Submitted forms are **read-only** and cannot be modified

- Users can **export submitted forms** (Web Only)
  - Formats: PDF (optionally CSV/Excel in future)

- Users can **sort forms**
  - Sort by: Name, Date
  - Order: ASC / DESC

- Users can **load more** forms as they scroll
